Format
<type>(scope): <description>

[optional body]

[optional footer(s)]
• Append ! after type/scope for BREAKING CHANGE
• Description: imperative mood, lowercase, no period
• Footer: BREAKING CHANGE: description or Refs: #123
Types
featNew feature (bumps MINOR)
fixBug fix (bumps PATCH)
docsDocumentation only
styleFormatting, whitespace
refactorNeither fix nor feature
perfPerformance improvement
testAdd/fix tests
buildBuild system/deps
ciCI/CD config
choreMaintenance tasks
revertRevert previous commit
Examples
feat(auth): add SSO login flow New feature with scope
fix(api): handle null response body Bug fix with scope
docs: update README setup steps No scope needed
feat(api)!: change auth token format Breaking change
refactor(ocr): simplify CSV pipeline Code refactor with scope
ci: add staging deploy workflow CI pipeline change
